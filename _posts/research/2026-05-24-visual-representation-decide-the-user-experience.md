같은 기능도 어떻게 보이느냐에 따라 완전히 다른 경험이 된다는 메모.

## 개요

- chat transcript로 보이는 agent와 task board로 보이는 agent는 같은 능력을 가져도 다르게 느껴진다.
- visual representation은 사용자가 시스템을 어떻게 감독하고 신뢰할지 결정한다.
- agent UX에서 시각화는 장식이 아니라 capability의 일부다.

## 핵심 질문

- 어떤 상태를 반드시 사용자에게 보여줘야 하는가?
- 진행 중인 작업, 위험, 변경 사항, interrupt 지점을 어떻게 표현해야 하는가?
- 정보량을 늘리지 않고 legibility를 높이는 표현은 무엇인가?

## 주장

모델은 무엇을 할 수 있는지를 결정하고, representation은 사용자가 그것을 이해하고 조종할 수 있는지를 결정한다. 그래서 visual representation은 user experience를 결정한다.

## 확장할 내용

- chat, diff, terminal, browser, board, timeline UI의 장단점
- agent progress visualization의 최소 단위
- confidence와 uncertainty를 UI로 표현하는 방법
